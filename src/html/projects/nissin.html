Responsive

I used media queries to make the content fit best the viewport size.

The menu changes from a bottom burger menu to a top bar menu, from mobile to desktop sizes.

Background images are added as the width changes to give to user the feeling that any size is the right size to view the website.

Performance

Performance is important and often left out. Especially when working with images and videos, if you don't have it in mind you can end up with a bloated and slow website.

I used WebP, srcset, and lazy loading.

If the browser can display WebP images it will, instead of the PNG (which is larger).

Srcset enables the browser to open a different images depending on the viewport width. This avoids loading unnecessarily large images.

Lazy loading allows the page to load images which are not visible by the user asynchronously and thus not blocking the rendering of the page.

BEM

BEM is a naming convention for CSS, it makes for more maintainable code through clear naming, flatter specificity and more modular code.
